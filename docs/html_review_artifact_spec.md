# HTML Review Artifact Spec

## Deliverables

Create:

- `out/html/lettera_fittizia_review.html`
- `out/html/lettera_fittizia_review_report.md`

Optional helper scripts may live in `tool/scripts/` if they make generation
repeatable.

## Page shape

The page is a static client review artifact.

Recommended layout:

- fixed/sticky left legend on desktop;
- paper preview on the right;
- on mobile, legend above the letter and the letter below;
- no external build step required;
- no real client content.

The preview should feel like an A4 page, not like a website section. It may
simulate the header/footer instead of embedding the real template assets, but it
must explicitly explain which parts are inherited from `Template_Vuoto.docx`.

## Fictional letter content

Use a fictional but realistic Bergamo Legal letter. It should exercise the main
sections:

- delivery method;
- date/place;
- recipient block;
- OGGETTO;
- opening;
- ordinary body paragraph;
- left subheading;
- centered ritual heading;
- list item;
- closing;
- signature block;
- optional attachments;
- optional postscript/disclaimer.

Do not mention real clients, real disputes, real personal data, or facts copied
from `previous_works/`.

## Required sections and legend items

Each item must have:

- `section id`;
- short label;
- matching color;
- short visible summary;
- full details shown on hover and focus;
- matching highlighted region in the letter.

Minimum section map:

| Section | Must show |
|---|---|
| Template header | inherited letterhead/logo/header, Garamond, not generated by body script |
| Delivery method | position, font, italic behavior, spacing |
| Date/place | right alignment, 12 pt, spacing |
| Recipient block | 8.5 cm indent, line handling, personal appellative mono-line rule |
| OGGETTO label | 16 pt, bold, default split/center behavior from current script |
| OGGETTO content | 12 pt, bold, justified content line |
| Opening | formal vs email behavior |
| Body paragraph | 12 pt Times New Roman, justified, spacing |
| Left subheading | 14 pt bold left, keep-with-next |
| Ritual heading | 16 pt bold center, keep-with-next |
| List item | 0.5 cm indent, justified, marker behavior |
| Closing | right aligned, spacing from body |
| Signature block | right aligned, bold names, spacing rules |
| Attachments | label and indented items |
| Template footer | inherited footer/registration/contact info |

## Formatting detail source

Use current implemented values from:

- `docs/regole_di_formattazione_in_lavorazione.md`
- `formatters/letters.py`

Use rationale and caveats from:

- `docs/regole_formattazione_lettere.md`
- `docs/client_approval_points.md`
- `feedback.md`
- `docs/dubbi_formattazione.md`

If these sources disagree, do not invent a resolution. In the HTML, show the
implemented rule as "current implementation" and note the conflict in
`out/html/lettera_fittizia_review_report.md`.

## Interaction rules

- Hover with mouse reveals full details.
- Keyboard focus reveals the same full details.
- The letter highlight and legend item should visually correspond.
- Use accessible contrast; highlights can be translucent but must remain visible.
- The page must still be understandable if printed or exported to PDF.

## Design rules

- Use a restrained law-firm style: clean typography, quiet colors, strong
  hierarchy, no decorative clutter.
- Avoid a marketing hero, cards-within-cards, decorative gradients, or stock
  imagery.
- Make text dense enough for review but not cramped.
- Keep colors distinct across adjacent sections.
- Use clear labels: the client must know what each highlight means without
  reading code.

## Acceptance checklist

Before finishing, verify:

- every legend item has a matching highlighted section;
- every highlighted section has a legend item;
- every legend item exposes full details on hover and focus;
- the fictional letter includes all required section types;
- the page opens directly as a static HTML file;
- no real client data appears;
- `previous_works/` and `assets/Template_Vuoto.docx` were not modified;
- the report lists sources used, unresolved conflicts, and client approval
  points.
